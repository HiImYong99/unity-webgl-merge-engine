# 🐾 애니멀 팝! 앱인토스 심사 제출 전 필수 체크리스트

앱 검토 요청(심사)을 제출하기 전에 반드시 아래 명시된 **고유 식별 정보**들을 실제 운영 데이터로 교체해야 합니다. 현재 코드에는 테스트용 ID가 포함되어 있어, 이대로 제출할 경우 결제 및 광고가 동작하지 않아 심 거절 사유가 될 수 있습니다.

---

## 1. 📢 광고 식별자 (Ad Group ID) 교체
사용자가 보상형 광고를 보고 부활하거나 2배속을 얻을 수 있도록, 발급받은 실제 광고 그룹 ID를 연동해야 합니다.

*   **수정 대상 파일:**
    *   `Assets/Plugins/WebBridge.jslib` (L.28, L.79)
    *   `Assets/WebGLTemplates/AnimalPop/index.html` (L.1181)
*   **변경 내용:**
    *   `REVIVE_AD_ID`: 토스 광고센터에서 생성한 **보상형 리워드 광고 ID**로 교체
    *   `SPEED_AD_ID`: (동일한 광고를 사용한다면 같은 ID 사용)

---

## 2. 💰 인앱 결제 상품 식별자 (SKU) 확인
'2배속 구매' 기능이 실제 토스페이 결제창과 연결되려면 SKU 값이 콘솔 설정과 정확히 일치해야 합니다.

*   **수정 대상 파일:**
    *   `Assets/WebGLTemplates/AnimalPop/index.html` (L.1388)
*   **변경 내용:**
    *   `SPEED_SKU`: 앱인토스 개발자 센터 [인앱 결제] 메뉴에 등록한 상품의 **SKU 코드** (예: `com.yourgame.speedboost`)와 대소문자까지 일치시켜주세요.

---

## 3. 🖼️ 리소스 프리 로딩 및 데이터 동기화
도감 사진이나 다음 동물 미리보기가 깨지지 않도록 최종 리소스 동기화 작업이 필요합니다.

*   **진행 단계:**
    1.  Unity 에디터 상단 메뉴: `AnimalPop` -> `Link Individual Sprites` 클릭 (사진 파일들이 `TemplateData/sprites/` 폴더로 자동 복사됩니다)
    2.  Unity 프로젝트 저장 후 다시 **WebGL 빌드** 실행 (사용 중인 템플릿이 `AnimalPop`인지 확인)

---

## 4. 📦 최종 .ait 패키지 생성 (Vite Build)
Unity 빌드 완료 후, 앱인토스 규격에 맞는 번들 파일을 새로 추출해야 합니다.

*   **진행 단계:**
    1.  터미널에서 `ait-build` 폴더로 이동: `cd ait-build`
    2.  빌드 명령어 실행: `npm run build`
    3.  생성된 `animal-pop.ait` 파일을 토스 디벨로퍼 콘솔에 업로드하여 최종 테스트 후 심사 요청 진행

---
